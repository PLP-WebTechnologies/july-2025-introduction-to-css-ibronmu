HTML5 Semantic Structure

Published on July 29, 2025

Why Semantic HTML Matters

Semantic HTML5 elements like <header>, <nav>, <main>, and <footer> provide meaning to web content, making it more accessible and SEO-friendly.

Key Semantic Elements

  • <article>: Independent, self-contained content
  • <section>: Thematic grouping of content
  • <aside>: Content indirectly related to main content
  • <figure>: Self-contained content like images or diagrams

Web Accessibility

Accessibility ensures that all users, including those with disabilities, can access web content.

Accessibility Best Practices

  1. Use proper heading hierarchy (h1-h6)
  2. Provide alt text for images
  3. Ensure sufficient color contrast
  4. Make interactive elements keyboard-navigable
  5. Use ARIA landmarks when appropriate

Search Engine Optimization

SEO helps search engines understand and rank your content appropriately.

Basic On-Page SEO Techniques

  • Use descriptive, keyword-rich page titles
  • Include meta descriptions
  • Structure content with proper heading hierarchy
  • Use semantic HTML5 elements
  • Optimize images with alt text
